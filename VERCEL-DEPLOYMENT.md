## üõ†Ô∏è Deployment to Vercel

### Prerequisites:
1. GitHub account
2. Vercel account (free): https://vercel.com/signup

### Step-by-Step Deployment:

#### Option 1: Deploy via Vercel Dashboard (Easiest)

1. **Push code to GitHub**:
```powershell
git add .
git commit -m "Add dynamic features with Vercel"
git push origin main
```

2. **Import to Vercel**:
   - Go to https://vercel.com/new
   - Click "Import Git Repository"
   - Select "JackAmichai/NewPersonalWebsite"
   - Click "Import"
   - Vercel will auto-detect settings
   - Click "Deploy"

3. **Done!** Your site is live at: `https://your-project.vercel.app`

#### Option 2: Deploy via Vercel CLI

1. **Install Vercel CLI**:
```powershell
npm install -g vercel
```

2. **Login to Vercel**:
```powershell
vercel login
```

3. **Deploy**:
```powershell
cd "c:\Users\yamichai\OneDrive - Deloitte (O365D)\Documents\General\webs\portfolio"
vercel --prod
```

4. **Follow prompts**:
   - Set up and deploy? Yes
   - Which scope? Your account
   - Link to existing project? No
   - Project name? jack-amichai-portfolio
   - Directory? ./
   - Override settings? No

5. **Done!** Note your deployment URL

## üåê Custom Domain Setup

1. **In Vercel Dashboard**:
   - Go to Project Settings ‚Üí Domains
   - Add your custom domain
   - Follow DNS instructions

2. **DNS Configuration**:
   - Add CNAME record: `www` ‚Üí `cname.vercel-dns.com`
   - Add A record: `@` ‚Üí Vercel IP

3. **SSL Certificate**: Auto-generated by Vercel

- Check build logs in Vercel Dashboard
- Verify all files are committed to Git
- Ensure `vercel.json` is present
